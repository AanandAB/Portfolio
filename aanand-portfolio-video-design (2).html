01 · Core Concept

The Big Idea: Video as the Universe

The video is not decoration. It IS the 3D world. Scroll position scrubs the video timeline. Each island the camera passes = one project reveal. An HTML/CSS overlay layer sits transparently on top — project cards, text, buttons appear and disappear in sync with what the camera is flying over.

🎬 The Core Mechanic: Video Timeline Scrubbing

The video element is set to paused and muted. JavaScript maps Lenis scroll progress (0 → 1) to video.currentTime (0 → 8s). The visitor "controls" the flythrough with their scroll wheel. This is cinematic, buttery, and requires zero 3D rendering — just a video tag and 30 lines of JS.

🎥
Video Scrubbing
// The Core Mechanism
Scroll controls video.currentTime. As you scroll, the camera flies forward. Island 1 appears at 1.5s, Island 2 at 3s, etc. Feels completely 3D but is just a video + CSS overlay.
📍
Island Hotspots
// HTML Overlay Anchors
Each island has a scroll-time "window" (e.g., 1.2s–2.4s). When currentTime enters that window, the island's project card fades in at the correct screen position. Click it = full project drawer opens.
✨
Neon Beam Roads
// The Navigation Metaphor
The neon beam "roads" already in the video serve as the visual scroll path. We reinforce this with a minimal HUD showing "Island 2 of 5 — approaching Theyyam." Progress = scroll progress.
⚡
Instant Load
// No 3D Overhead
The video is your heaviest asset (~6MB). Compress to WebM for 60% smaller (~2.4MB). Load a poster frame JPG first. Video preloads behind it. Total TTFI under 1.5s. No WebGL, no shaders needed.
02 · Video Scroll Architecture

How the Scrubbing Works

The entire scroll experience is driven by 3 interconnected systems: Lenis smooth scroll → GSAP scroll progress → video.currentTime. Here's exactly how to implement it.

// ─── 1. SETUP: Lenis + video reference ─────────────────────── import Lenis from '@studio-freight/lenis' import gsap from 'gsap' import { ScrollTrigger } from 'gsap/ScrollTrigger' const lenis = new Lenis({ lerp: 0.06, smoothWheel: true }) const video = document.getElementById('world-video') video.pause() video.currentTime = 0 // ─── 2. CORE: map scroll → video time ──────────────────────── // Page height = 800vh. Video = 8 seconds. // 100vh of scroll = 1 second of video. gsap.registerPlugin(ScrollTrigger) ScrollTrigger.create({ trigger: '#scroll-container', start: 'top top', end: 'bottom bottom', scrub: true, onUpdate(self) { video.currentTime = self.progress * video.duration } }) // ─── 3. ISLANDS: trigger overlays at exact timestamps ──────── const islands = [ { id: 'cafe', enter: 0.8, exit: 2.0, x: '50%', y: '55%' }, // CafePOSPro { id: 'theyyam', enter: 2.2, exit: 3.4, x: '28%', y: '45%' }, // Theyyam App { id: 'knightly',enter: 3.5, exit: 4.8, x: '68%', y: '50%' }, // Web3 Chess { id: 'evently', enter: 5.2, exit: 6.6, x: '50%', y: '48%' }, // Evently { id: 'bytebot', enter: 7.0, exit: 8.0, x: '38%', y: '40%' }, // Bytebot AI ] // Lenis RAF loop function raf(time) { lenis.raf(time) requestAnimationFrame(raf) } requestAnimationFrame(raf)

💡 Critical Design Insight

Because the video already provides cinematic depth, colour, and drama, your HTML overlay should be minimal and clean. Resist adding heavy CSS effects on top of the video — let the video breathe. White/light text on dark glassmorphism cards. One accent colour per island matching the video's existing neon (cyan for Cafe/Knightly, gold for Theyyam, pink for Evently, green for Bytebot).

Build the useVideoTime and useIslandVisible hooks

Cursor Chat Prompt — New files in src/hooks/
Create two custom hooks in src/hooks/: 1. useScrollProgress.js - Uses GSAP ScrollTrigger to track scroll progress of the element with id="scroll-container" - Returns a ref-based value (using useRef, not useState — avoid re-renders) named scrollProgress that goes from 0 to 1 - Also returns videoDuration (default 8) and currentTime (scrollProgress * videoDuration) 2. useIslandVisible.js - Accepts (currentTime, island) where island has videoEnter and videoExit seconds - Returns { isVisible: boolean, progress: 0–1 within the island's time window } - isVisible is true when currentTime >= island.videoEnter && currentTime <= island.videoExit - progress = (currentTime - island.videoEnter) / (island.videoExit - island.videoEnter), clamped 0–1 Export both as named exports.

Pro Tips Getting the Best Out of Cursor
Always open the relevant file

Before writing a prompt, open the file you want Cursor to edit (Cmd+P). Cursor Chat with a file open automatically uses that file as context.

Use Cmd+K for inline edits

For small changes (tweaking a colour, fixing a bug), use Cmd+K inline edit instead of Chat. It's faster and more precise for targeted changes.

When it gets confused, reset context

If Cursor starts giving wrong code, start a new Chat window (Cmd+Shift+L) and paste only the specific file + a clear single question.

Reference files with @

In Cursor Chat, type @islands.js to attach that file to your prompt. Always attach the files most relevant to your request.

Fix scroll jank immediately

If video scrubbing feels jerky: reduce Lenis lerp to 0.04, add gsap.ticker.lagSmoothing(0), and ensure GSAP ticker drives Lenis (not requestAnimationFrame separately).

Test island timing by hand

The island video timestamps (0.8s, 2.2s etc.) are estimates. After the first build, manually scrub the video in browser console: document.querySelector('video').currentTime = 1.5 and adjust until cards appear centred on each island.

Don't ask Cursor for Three.js

The .cursorrules file prevents this, but if Cursor ever suggests Three.js or WebGL for anything, reject it and add "do not use Three.js or WebGL" to your prompt.

Generate more islands with AI video tools

Need a Salesforce-themed island? Use Kling AI or Runway with the prompt: "cinematic flythrough of a neon-lit floating island in dark space, cloud city architecture, blue glow, same style as [attach frame from your video]."
